# บทที่ 1: บทนำ (Chapter 1. Introduction)

## เหตุผลในการทำเหมืองข้อมูล (Why Data Mining?)
ข้อมูลในโลกเพิ่มขึ้นอย่างรวดเร็วจากหลายแหล่ง เช่น ธุรกิจออนไลน์, วิทยาศาสตร์, และสื่อดิจิทัล ทำให้เกิดสภาวะ *“ข้อมูลท่วม แต่ความรู้ไม่พอใช้”*  
Data Mining จึงถูกพัฒนาขึ้นเพื่อช่วยวิเคราะห์และดึงรูปแบบที่มีประโยชน์จากชุดข้อมูลขนาดใหญ่ได้อย่างอัตโนมัติ

---

## การทำเหมืองข้อมูลคืออะไร (What Is Data Mining?)
เป็นกระบวนการค้นหา **รูปแบบหรือความรู้ที่สำคัญ** จากข้อมูลจำนวนมาก ซึ่งมักไม่ชัดเจนหรือไม่เคยถูกค้นพบมาก่อน  
มีชื่อเรียกใกล้เคียงกัน เช่น KDD, Knowledge Extraction, Pattern Analysis และ Business Intelligence

---

## กระบวนการ KDD (Knowledge Discovery Process)
ประกอบด้วยขั้นตอนสำคัญ:
1. Data Cleaning  
2. Data Integration  
3. Data Selection  
4. Data Transformation  
5. Data Mining  
6. Pattern Evaluation  
7. Knowledge Presentation  

โดยแต่ละสาขามองต่างกันเล็กน้อย เช่น ML เน้น Pre-processing, BI เน้นการใช้ผลลัพธ์เพื่อสนับสนุนการตัดสินใจ

---

## ขอบเขตของงานเหมืองข้อมูล (Multi-Dimensional View)
- **ข้อมูลที่ทำเหมืองได้**: ฐานข้อมูล, ข้อมูลธุรกรรม, เซนเซอร์/สตรีม, ข้อความ, เว็บ, เครือข่ายสังคม  
- **รูปแบบที่ค้นหา**: การสรุปข้อมูล, ความสัมพันธ์, การจำแนกประเภท, การจัดกลุ่ม, ค่าผิดปกติ, ลำดับเวลา  
- **เทคโนโลยีที่เกี่ยวข้อง**: ML, สถิติ, OLAP, Pattern Recognition, Visualization  
- **งานประยุกต์**: การตลาด, การเงิน, การแพทย์, เครือข่ายสังคม, ระบบแนะนำข้อมูล

---

## ฟังก์ชันหลักของการทำเหมืองข้อมูล (Data Mining Functions)

| ฟังก์ชัน | คำอธิบาย |
|---------|-----------|
| **Generalization** | สรุปและอธิบายข้อมูลหลายมิติ เช่น Characterization |
| **Pattern Discovery** | หาความถี่และความสัมพันธ์ เช่น Association Rules |
| **Classification** | สร้างโมเดลทำนายคลาส เช่น Decision Trees, SVM |
| **Clustering** | แบ่งข้อมูลเป็นกลุ่มโดยไม่ใช้ label |
| **Outlier Analysis** | ค้นหาค่าหรือเหตุการณ์ที่ผิดปกติ |
| **Time & Sequential Patterns** | วิเคราะห์แนวโน้มและลำดับเหตุการณ์ |
| **Structure/Network Mining** | วิเคราะห์ข้อมูลแบบกราฟและเครือข่าย |

---

## ประเด็นที่ท้าทาย (Major Issues)
- จัดการข้อมูลที่หลากหลายและซับซ้อน  
- อัลกอริทึมต้องรองรับข้อมูลขนาดใหญ่  
- ความเป็นส่วนตัวและผลกระทบทางสังคม  
- การทำเหมืองแบบออนไลน์/สตรีม  
- การนำความรู้ที่ได้ไปใช้งานอย่างเหมาะสม

---

## สรุป
Data Mining เป็นกระบวนการดึงความรู้จากข้อมูลขนาดใหญ่ ผ่านขั้นตอนใน KDD และใช้ได้กับข้อมูลหลากหลายรูปแบบ  
มีฟังก์ชันสำคัญ เช่น การจำแนก การจัดกลุ่ม และการหาความสัมพันธ์ และมีบทบาทสำคัญในหลายอุตสาหกรรม
